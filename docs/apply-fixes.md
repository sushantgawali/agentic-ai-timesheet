# /timesheet:apply-fixes

## Overview

Applies the correction plan generated by `/timesheet:propose-fixes` directly
to `data/kimai_timesheets.csv`. Always creates a timestamped backup first.
By default only high-confidence AUTO-APPLY changes are written; pass
`--include-review-items` to also apply NEEDS_REVIEW defaults and add missing
entries.

---

## Usage

```
/timesheet:apply-fixes
```

Apply only AUTO-APPLY and OVERLAP FIX changes (safe, high-confidence).

```
/timesheet:apply-fixes --include-review-items
```

Also applies NEEDS_REVIEW defaults and inserts missing-entry rows detected
from Slack/git activity.

## What gets applied

### Default mode (no flag)

| Source check | Change written |
|---|---|
| CHECK-1 | Fix `end` timestamps with hour ≥ 24; recalculate `hours` |
| CHECK-2 | Delete catch-all overlap rows; trim partial overlaps |
| CHECK-4/5 | Recode archived project to user's primary active project |
| CHECK-6 | Correct `hourly_rate` to canonical value |
| CHECK-7 | Infer and fill `activity` from description keywords (HIGH confidence only) |
| CHECK-8 | Insert `[No description — please update]` when both `activity` and `project` are also missing |
| CHECK-9 | Fill `project` when user has exactly one assignment |

### With `--include-review-items`

Everything above, plus:

| Source check | Change written |
|---|---|
| CHECK-3 | Delete entries on approved leave days |
| CHECK-7 | Set uninferable activities to `"General"` |
| CHECK-8 | Fill description as `"{activity} work on {project}"` |
| CHECK-9 | Fill missing project (multiple assignments → first alphabetically) |
| CHECK-10 | Keep deactivated employee entries (default A) |
| CHECK-12 | Insert new rows for active days with no timesheet |

## Implementation

### Safety: backup

Before any write, the original file is copied to:
```
data/kimai_timesheets.backup.YYYYMMDD-HHMMSS.csv
```

To restore:
```bash
cp data/kimai_timesheets.backup.<timestamp>.csv data/kimai_timesheets.csv
```

### Processing order

1. Field-level fixes (CHECK-1, 4/5, 6, 7, 8, 9) applied in a single pass over rows
2. Overlap fixes (CHECK-2) applied in a second pass after field fixes, so
   trimmed timestamps from CHECK-1 are already clean
3. Deletions collected during both passes and applied at write time (not in-place)
4. New rows (CHECK-12) appended after all existing rows

### Post-write validation

After writing, the output file is re-read and checked for:
- Any row with an empty `user` field
- Any remaining `end` timestamp with hour ≥ 24

Failures are printed as warnings but do not roll back the file.

### Change log format

Each entry in the change log is prefixed with one of:

| Prefix | Meaning |
|--------|---------|
| `CHANGED` | A field value was updated in an existing row |
| `DELETED` | An existing row was removed |
| `ADDED` | A new row was inserted |
